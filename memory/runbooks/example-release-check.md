# Starter Example: Release Check

This is a starter example for a runbook note.
Replace or delete it once the repository has a real operator procedure worth keeping.

## Load when

- A change touches shipped surfaces and you need one compact verification pass before closing the work.

## Checks

1. Refresh the installed payload through the canonical upgrade path.
2. Run the narrowest package or workspace proof lane that covers the changed surface.
3. Confirm any generated mirrors still match their canonical source.

## Review when

- The release, upgrade, or generated-surface verification path changes.

## Failure signals

- A shipped surface changes without a matching payload refresh or mirror check.

## Verify

- The intended proof lane passes and no stale generated-surface drift remains.
