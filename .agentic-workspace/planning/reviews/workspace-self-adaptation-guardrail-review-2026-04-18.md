# Workspace Self-Adaptation Guardrail Review

## Purpose

Apply the compact workspace-self-adaptation guardrail to recent repo examples so the guardrail is anchored in ordinary dogfooding rather than policy prose alone.

## Examples

1. Planning recovery question set and meaning-boundary tightening:
   - classification: workspace adaptation
   - why: the friction was in workspace-owned recovery and contract surfaces, and one bounded clarification reduced rereads without hiding a repo-owned structural problem

2. Optimization-bias visibility in recovery and reporting:
   - classification: workspace adaptation
   - why: the workspace was missing a compact way to expose an already-existing repo policy, so the cheapest honest fix was to improve the workspace surface itself

3. Planning friction as repo-friction evidence:
   - classification: surface repo friction
   - why: once the workspace already exposes planning recovery and proof contracts, repeated seam, tranche-boundary, or ownership ambiguity should stay visible as repo-friction evidence instead of being hidden behind more compensating workspace logic

## Conclusion

The compact guardrail is strong enough for the current maturity level:

- adapt inside Agentic Workspace when the friction is really about workspace fit or contract clarity
- keep repo-friction evidence visible when the root problem is repo-owned structure
- reject narrow compensations that would accumulate into a concealment layer
